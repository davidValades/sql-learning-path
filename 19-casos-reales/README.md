## 19.0 Estado heredado del modelo

Tu base ya incluye:

- Diseño y evolución estructural (DDL+DML).
- Consultas analíticas y objetos (JOINs, subconsultas, vistas).
- Rendimiento, seguridad y automatización (índices, DCL, PL/SQL).

No empiezas de cero: partes de una base que evolucionó tema a tema.

---

## 19.1 Contexto de negocio

La empresa unifica tres áreas:

- **Ventas** (e-commerce).
- **Operaciones asistenciales** (hospital).
- **Logística** (aerolínea).

Dirección necesita un sistema consolidado para auditoría, reporting y operación diaria.

---

## 19.2 Flujo de entrega en entorno profesional

1. Levantar requerimientos funcionales y técnicos.
2. Definir cambios de esquema y plan de migración.
3. Implementar scripts versionados.
4. Ejecutar pruebas funcionales y de rendimiento.
5. Publicar dashboard operativo con vistas seguras.
6. Entregar runbook de operación y rollback.

---

## 19.3 Criterios de calidad del capstone

- Integridad referencial intacta.
- Transacciones críticas con control de errores.
- Permisos mínimos por rol.
- Consultas clave por debajo del SLA acordado.
- Evidencias de auditoría para cambios sensibles.

---

## 19.4 Evidencias mínimas

- Script maestro de despliegue.
- Script de rollback.
- Consultas de validación.
- Informe de rendimiento antes/después.
- Matriz de permisos por usuario/rol.
