## 16.0 Estado heredado del modelo

Venimos de seguridad (Tema 14) con usuarios y roles activos. Ahora añadimos lógica interna manteniendo control transaccional:

- Validaciones de stock.

## 16.1 Bloques anónimos y variables

```sql
DECLARE
  v_total_pedidos NUMBER;
  v_total_ventas NUMBER(12,2);
BEGIN
  SELECT COUNT(*), NVL(SUM(total),0)
  INTO v_total_pedidos, v_total_ventas
  FROM pedidos;

  DBMS_OUTPUT.PUT_LINE('Pedidos: ' || v_total_pedidos);
  DBMS_OUTPUT.PUT_LINE('Ventas: ' || v_total_ventas);
END;
/
```

## 16.2 Procedimientos para operaciones de negocio

```sql
CREATE OR REPLACE PROCEDURE sp_registrar_pedido (
  p_id_pedido   IN NUMBER,
  p_id_cliente  IN NUMBER,
  p_id_producto IN NUMBER,
  p_cantidad    IN NUMBER
) AS
  v_precio productos.precio%TYPE;
  v_stock  productos.stock%TYPE;
BEGIN
  SELECT precio, stock INTO v_precio, v_stock
  FROM productos
  WHERE id_producto = p_id_producto
  FOR UPDATE;

  IF p_cantidad <= 0 THEN
    RAISE_APPLICATION_ERROR(-20001, 'Cantidad inválida');
  ELSIF v_stock < p_cantidad THEN
    RAISE_APPLICATION_ERROR(-20002, 'Stock insuficiente');
  END IF;

  INSERT INTO pedidos (id_pedido, id_cliente, id_producto, cantidad, fecha_pedido, total)
  VALUES (p_id_pedido, p_id_cliente, p_id_producto, p_cantidad, SYSDATE, v_precio * p_cantidad);

  UPDATE productos
  SET stock = stock - p_cantidad
  WHERE id_producto = p_id_producto;
END;
/
```

## 16.3 Funciones reutilizables

```sql
CREATE OR REPLACE FUNCTION fn_riesgo_stock (p_id_producto IN NUMBER)
RETURN VARCHAR2
AS
  v_stock productos.stock%TYPE;
BEGIN
  SELECT stock INTO v_stock
  FROM productos
  WHERE id_producto = p_id_producto;

  RETURN CASE
    WHEN v_stock <= 5 THEN 'CRITICO'
    WHEN v_stock <= 20 THEN 'BAJO'
    ELSE 'NORMAL'
  END;
END;
/
```

## 16.4 Triggers de auditoría

```sql
CREATE TABLE auditoria_stock (
  id_auditoria NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  id_producto NUMBER NOT NULL,
  stock_anterior NUMBER,
  stock_nuevo NUMBER,
  fecha_cambio DATE DEFAULT SYSDATE,
  usuario_bd VARCHAR2(100)
);

CREATE OR REPLACE TRIGGER trg_auditoria_stock
AFTER UPDATE OF stock ON productos
FOR EACH ROW
BEGIN
  INSERT INTO auditoria_stock (id_producto, stock_anterior, stock_nuevo, usuario_bd)
  VALUES (:OLD.id_producto, :OLD.stock, :NEW.stock, USER);
END;
/
```

> Esta tabla se usará en Tema 17 para optimizar consultas de auditoría.

## 16.5 Paquetes para organizar lógica

```sql
CREATE OR REPLACE PACKAGE pkg_operaciones AS
  PROCEDURE recalcular_total_pedido(p_id_pedido IN NUMBER);
END pkg_operaciones;
/

CREATE OR REPLACE PACKAGE BODY pkg_operaciones AS
  PROCEDURE recalcular_total_pedido(p_id_pedido IN NUMBER) IS
  BEGIN
    UPDATE pedidos p
    SET total = p.cantidad * (
      SELECT pr.precio FROM productos pr WHERE pr.id_producto = p.id_producto
    )
    WHERE p.id_pedido = p_id_pedido;
  END;
END pkg_operaciones;
/
```
